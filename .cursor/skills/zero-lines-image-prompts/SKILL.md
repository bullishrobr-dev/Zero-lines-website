# Zero Lines Image Prompts

## Purpose
Use this skill when the user needs polished image-generation prompts for Zero Lines.

This skill should produce:
- a strong main prompt
- optional prompt variations
- a negative prompt when useful
- concise art-direction notes
- outputs that stay commercially usable

## Brand Rules
Default to:
- clinical-luxury skincare
- advanced science credibility
- premium tech minimalism
- elegant restraint
- product-first composition

Keep the Zero Lines visual world aligned with:
- aqua or Tiffany-inspired accents
- frosted or translucent packaging
- charcoal or black typography
- calm, precise lighting
- premium materials such as glass, water, stone, and brushed metal

## Prompt Workflow
1. State the objective.
2. List the available inputs.
3. State the working assumptions.
4. Write one best main prompt.
5. Add 3 variations when requested or useful.
6. Add a negative prompt when useful.
7. Add short art-direction notes.

## Prompt Construction Rules
Include, when relevant:
- product type
- packaging finish
- environment or surface
- lighting style
- framing
- palette
- mood
- commercial intent

Prefer prompts that feel:
- premium
- specific
- visually controlled
- immediately usable for image models

## Output Formats
### Standard prompt request
Return:
1. Objective
2. Available inputs
3. Assumptions
4. Main prompt
5. 3 variations
6. Negative prompt
7. Art direction notes
8. Best use case

### Quick prompt request
Return:
1. Main prompt
2. Negative prompt
3. One-line usage note

## Channel Notes
### E-commerce
- neutral or refined controlled background
- clean silhouette readability
- premium lighting
- realistic, conversion-friendly detail

### Editorial
- more atmosphere
- stronger composition
- still restrained and brand-safe

### Campaign
- emotional but controlled
- broader sense of world-building
- luxury over spectacle

### Educational science
- clean, elegant, not cartoonish
- precise structural storytelling
- clinical beauty, not cold lab graphics

## Limited Inputs Rule
If only one mockup or product name is provided:
- infer carefully from the current family
- state assumptions briefly
- still give a usable first-pass prompt
